## 目标
- 在不修改业务代码的前提下，验证 FastAPI 后端能在本机正常启动并接受请求

## 现状概览
- 应用结构：`backend/api.py` 定义 FastAPI 实例 `app` (`c:\Users\admin\Desktop\ResumeAssistant\backend\api.py:19`)，暴露 `POST /run` SSE 接口 (`c:\Users\admin\Desktop\ResumeAssistant\backend\api.py:68`).
- 业务入口：`EditorAgent` 在模块加载时实例化 OpenAI 客户端 (`c:\Users\admin\Desktop\ResumeAssistant\backend\editor_agent.py:48-53`)，但不在启动阶段发网请求。
- 依赖清单：`requirements.txt` 包含 `fastapi`/`uvicorn`/`openai` 等 (`c:\Users\admin\Desktop\ResumeAssistant\backend\requirements.txt:18-20`).
- CORS：允许前端 `http://localhost:5173` (`c:\Users\admin\Desktop\ResumeAssistant\backend\api.py:22-28`).

## 执行步骤
1. 进入后端目录：`cd c:\Users\admin\Desktop\ResumeAssistant\backend`
2. 准备虚拟环境（若已存在 `.venv` 可复用）：
   - 复用：`Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass; .\.venv\Scripts\Activate.ps1`
   - 或新建：`python -m venv .venv` 后再激活
3. 安装依赖：`pip install -r requirements.txt`
4. 启动服务（开发模式热重载）：`uvicorn api:app --host 0.0.0.0 --port 8000 --reload`
   - 期望日志出现 `Uvicorn running on http://0.0.0.0:8000` 和 `Application startup complete`。

## 验证方式
- 健康性（基本连通）：浏览器或 `curl` 访问 `http://localhost:8000/docs`，应看到 FastAPI 自动文档页面。
- 接口可用性（SSE）：
  - 发送最小请求：
    - `curl -N -X POST -H "Content-Type: application/json" -d "{\"input\": \"你好\"}" http://localhost:8000/run`
  - 期望返回以 `data: { ... }` 形式的 SSE 流；若外部模型调用失败，守卫器会返回 `错误: ...` 文本，但服务本身仍正常工作（`c:\Users\admin\Desktop\ResumeAssistant\backend\api.py:40-66`).

## 不改业务逻辑保证
- 仅进行环境与运行验证，不变更任何 `.py` 源文件和工具逻辑。
- 若依赖缺失或端口占用，仅做运行参数/环境层面的调整（如换端口），不改代码。

## 可能风险与应对
- 端口冲突：改用 `--port 8001` 启动并在验证步骤中同步替换 URL。
- 依赖安装失败（Windows 构建问题）：逐项安装并记录具体报错包名，必要时锁定兼容版本（仍不改业务代码）。
- 外网不可达导致 `/run` 内部模型流报错：属外部服务问题；只要 SSE 能返回（哪怕是错误消息），即可判定后端启动与路由工作正常。
