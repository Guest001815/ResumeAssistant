# 面试突击：核心亮点提炼（修正版）

非常好的直觉！您抓住了重点。**Prompt Engineering（提示词工程）** 确实是本项目比 SSE 更核心、更具技术含量的亮点。SSE 只是传输手段，而 Prompt 设计才是赋予 Agent "灵魂" 的关键。

## 修正后的三大核心亮点

### 亮点一：Orchestrator 编排架构 (架构层)
- **核心词**: **解耦、调度、扩展性**。
- **话术**: "我设计了一个中心化的编排器，把业务逻辑从 API 层剥离。它像一个指挥家，根据当前状态动态调度 Plan/Guide/Editor 三个专职 Agent，保证了系统的可维护性和扩展性。"

### 亮点二：LLM 驱动的确定性状态机 (逻辑层)
- **核心词**: **结构化输出、流程控制、防幻觉**。
- **话术**: "为了解决 AI 不可控的问题，我强制 LLM 输出 JSON 格式的 Action 指令（如 `REQUEST_CONFIRM`）。用代码逻辑去约束 AI 的行为，实现了'灵活性'与'严谨性'的平衡。"

### 亮点三：动态上下文 Prompt 工程 (智能层) —— *重点替换*
- **核心词**: **动态注入、状态感知、策略切换**。
- **话术**: "我的 Agent 不是傻乎乎地每次都用同一套 Prompt。我实现了一套**动态 Prompt 构建机制**：
    1.  **状态感知**：根据当前是'挖掘信息'还是'确认草稿'，动态切换 System Prompt。
    2.  **上下文注入**：把用户刚刚跳过的任务、已完成的任务动态塞进 Prompt，让 AI 知道'用户刚才干了什么'，从而生成极具同理心的回复（比如'既然你跳过了上一个，那我们轻松点聊这个...'）。
    3.  **策略分支**：针对学生项目和真实项目，动态加载不同的引导策略。"

## 极速理解指南 (装修公司比喻 - 强化版)
- **API**: 前台。
- **Orchestrator**: 项目经理。
- **Prompt Manager (亮点三)**: **"话术培训师"**。
    - 他手里拿着好几本剧本（不同状态的 Prompt）。
    - 进门前，他会看一眼客户刚才是不是发火了（跳过任务），然后塞给沟通专员（Guide Agent）一本特定的剧本：“客户刚跳过一个任务，你这次说话客气点，别问太细。”
    - 这就是**动态上下文**的魔力！

## 交付形式
- **面试小抄 (Cheat Sheet)**: 针对这三个亮点，准备"面试官可能会问的深层问题"及参考答案。
- **Prompt 动态构建图解**: 一张图看懂 Prompt 是怎么"拼"出来的。
