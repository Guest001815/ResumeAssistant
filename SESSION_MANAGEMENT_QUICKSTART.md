# 会话管理系统 - 快速开始

## 🎉 新功能介绍

现在支持**无登录多会话管理**！您可以：
- 📁 管理多个简历优化项目（不同职位、不同简历）
- 🔄 快速切换不同的优化场景
- 💾 自动保存，刷新页面不丢失进度
- ✏️ 自定义会话名称
- 🎨 简洁的抽屉式UI

---

## 🚀 快速开始

### 1. 启动服务

**后端**：
```bash
cd backend
python api.py
```

**前端**：
```bash
cd web
npm run dev
```

### 2. 创建第一个会话

1. 访问 http://localhost:5173
2. 上传简历PDF
3. 输入职位描述（JD）
4. 点击"开始优化"

系统会自动：
- 提取公司和职位信息
- 生成会话名称（如："张三 - 前端工程师 - 字节跳动"）
- 开始分析简历

### 3. 使用会话管理

**打开会话列表**：
- 桌面端：点击左上角 `☰` 按钮
- 移动端：点击顶部 `☰` 按钮

**切换会话**：
1. 打开会话列表
2. 悬停到要切换的会话（会展开详情）
3. 点击"切换"按钮

**新建会话**：
- 点击会话列表右上角 `+` 按钮
- 或点击 `🏠` 返回首页重新上传

**重命名会话**：
1. 悬停到会话卡片
2. 点击 `✏️` 图标
3. 输入新名称，按回车确认

**删除会话**：
1. 悬停到会话卡片
2. 点击 `🗑️` 图标
3. 确认删除

## 💡 使用技巧

### 1. 针对不同公司优化简历

为不同公司的同一职位创建多个会话：
```
会话1: 张三 - 前端工程师 - 字节跳动
会话2: 张三 - 前端工程师 - 阿里巴巴  
会话3: 张三 - 前端工程师 - 腾讯
```

每个会话独立优化，互不影响！

### 2. 同一简历应聘不同职位

```
会话1: 李四 - 前端工程师 - 字节跳动
会话2: 李四 - 全栈工程师 - 字节跳动
会话3: 李四 - 算法工程师 - 字节跳动
```

### 3. 使用自定义名称

将会话重命名为更有意义的名称：
```
原名称: 张三 - 前端工程师 - 字节跳动
新名称: 字节 - 前端 - 已投递 ✅
```

### 4. 会话状态管理

利用emoji标记会话状态：
```
✅ 字节 - 前端 - 已投递
📝 阿里 - 后端 - 修改中
⏳ 腾讯 - 算法 - 待投递
✓ 美团 - 全栈 - 已完成
```

## 📊 会话卡片信息

每个会话显示：
```
┌────────────────────────────┐
│ ✅ 📄                      │
│ 会话名称              ✏️   │
│                            │
│ 📝 简历文件名              │
│ 💼 职位名称                │
│ 📊 3/10 任务              │
│ 🕒 2小时前                │
│                            │
│ [切换]  🗑️                │
└────────────────────────────┘
```

图标说明：
- ✅ = 当前会话
- 📄 = 普通会话
- ✓ = 已完成会话
- 📝 = 简历文件
- 💼 = 职位
- 📊 = 任务进度
- 🕒 = 更新时间

## 🔧 常见问题

### Q: 会话数据存储在哪里？
A: 
- **LocalStorage**: 会话列表、消息历史（快速访问）
- **服务器磁盘**: 完整会话数据（持久化）
- 位置：`backend/storage/sessions/`

### Q: 清除浏览器缓存会丢失数据吗？
A: 
- 会话本身**不会丢失**（存储在服务器）
- 消息历史**会丢失**（仅存在LocalStorage）
- 刷新页面会从服务器重新加载会话

### Q: 可以创建多少个会话？
A: 理论上无限制，但建议：
- 日常使用：5-10个活跃会话
- 定期清理已完成的会话

### Q: 如何备份会话数据？
A: 备份 `backend/storage/sessions/` 目录即可

### Q: 支持导出会话吗？
A: 当前版本不支持，计划在未来版本添加

## 🎯 最佳实践

   - 添加emoji标记状态
